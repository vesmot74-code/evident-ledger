EVIDENT LEDGER

Deterministic Event Ledger

Криптографическое доказательство фиксации файла во времени

Evident Ledger — локальная система фиксации цифровых объектов: SHA-256 хеширование, hash-linked цепочка событий, опциональная внешняя временная метка RFC 3161 (TSA) и независимая офлайн-верификация. Не облачный сервис — вы полностью контролируете свои данные локально.

Зачем это нужно

  • Доказательство существования файла в момент времени — если позже возникнет спор о том, менялся ли документ, у вас есть криптографически проверяемая цепочка событий с merkle-доказательством.
  • Независимая проверка — верификация (evident-verify) работает офлайн, не требует доверия к серверу: пересчитывает merkle root, проверяет подпись, сверяет файл с зафиксированным хэшем.
  • Локальный контроль целостности — GUI дополнительно проверяет, что файлы на вашем диске совпадают с зафиксированными хэшами, обнаруживая случайную или намеренную подмену уже после фиксации.

Когда это пригодится

  • Фиксация версии документа перед отправкой на согласование/подпись
  • Доказательство даты создания артефакта (черновик, договор, чертёж)
  • Аудиторский след изменений в проекте с восстановимой историей
  • Локальная альтернатива, когда нельзя/не хочется отправлять файл в облачный сервис

Как это работает

  1. Commit — файл хешируется (SHA-256) локально, событие отправляется в цепочку
  2. Anchor — сервер назначает sequence, строит merkle root, подписывает результат; опционально запрашивает TSA-метку у FreeTSA
  3. Verify — proof можно проверить в любой момент: через GUI (backend + локальная сверка файла) или офлайн через CLI
