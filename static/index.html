Deterministic Event Ledger

Криптографическое доказательство фиксации файла во времени

Evident Ledger — локальная система фиксации цифровых объектов: SHA-256 хеширование, hash-linked цепочка событий, опциональная внешняя временная метка RFC 3161 (TSA) и независимая офлайн-верификация. Не облачный сервис — вы полностью контролируете свои данные локально.

Два независимых компонента

Локальный клиент и серверный ledger решают разные задачи и не заменяют друг друга.

Локальный клиент

evident (CLI / GUI)

  • хеширует файл (SHA-256) локально
  • отправляет событие в цепочку
  • офлайн-верификация proof, без обращения к серверу

Работает локально. Файл никогда не отправляется на сервер.

Серверный ledger

Evident Ledger Server

  • назначает sequence, строит Merkle root
  • подписывает результат (Ed25519)
  • опционально запрашивает RFC 3161 TSA-метку у FreeTSA
  • выдаёт независимо проверяемый сертификат (SAC)

Модель доверия

Доверие строится на:

  • криптографическом хешировании (SHA-256)
  • hash-linked цепочке событий с Merkle-доказательством
  • внешнем источнике времени (RFC 3161 TSA)
  • возможности независимой офлайн-верификации

Зачем это нужно

Доказательство во времени

Если позже возникнет спор о том, менялся ли документ — у вас есть криптографически проверяемая цепочка событий с Merkle-доказательством.

Независимая проверка

evident-verify работает офлайн, не требует доверия к серверу: пересчитывает Merkle root, проверяет подпись, сверяет файл с зафиксированным хэшем.

Hash-only приватность

Можно узнать, зафиксирован ли файл в системе, передав на сервер только его SHA-256, без загрузки самого документа.

Как это работает

Шаг 1

Commit

Файл хешируется (SHA-256) локально, событие отправляется в цепочку.

Шаг 2

Anchor

Сервер назначает sequence, строит Merkle root, подписывает результат; опционально запрашивает TSA-метку у FreeTSA.

Шаг 3

Verify

Proof можно проверить в любой момент: через GUI (backend + локальная сверка файла) или офлайн через CLI.

Когда это пригодится

Фиксация версии документа перед отправкой на согласование или подпись
Доказательство даты создания артефакта — черновик, договор, чертёж
Аудиторский след изменений в проекте с восстановимой историей
Локальная альтернатива, когда файл нельзя отправлять в облачный сервис

Уровни защищённости доказательств

Evident Ledger спроектирован как платформа с прогрессивной защищённостью доказательств. Каждый следующий уровень добавляет ещё один криптографический или операционный механизм защиты, не изменяя сами зафиксированные данные.

Бесплатно

Базовый

SHA-256 фиксация, hash-linked цепочка событий, машинная временная метка (FreeTSA, RFC 3161).

Временная метка предназначена преимущественно для технической проверки. Её доказательная ценность зависит от применимого законодательства.

Уровень 2

Юридический TSA

Всё из Базового + временная метка от квалифицированного поставщика TSA, признанного в выбранной юрисдикции.

Обеспечивает независимо проверяемую метку времени, признаваемую применимым законодательством там, где это применимо.

Уровень 3

Резервируемый аудит

Всё из уровня 2 + зашифрованная резервная копия аудиторского журнала на сервере компании.

Повышает устойчивость к потере локальных данных и обеспечивает непрерывность аудита.

Уровень 4

Персонализированная подпись

Всё из уровня 3 + собственная пара ключей (приватный/публичный) для пользователя.

Криптографическая подпись пользователя усиливает атрибуцию, позволяя проверять доказательства независимо от оператора платформы.

Каждый уровень усиливает техническую защищённость и доказательную устойчивость зафиксированных данных. Юридическая допустимость и доказательственный вес материалов определяются применимым законодательством, процессуальными правилами и решением суда либо соответствующего регулятора.
