# Contributing to Evident Ledger

Thank you for your interest in contributing.

## Development Workflow

1. Fork the repository.
2. Create a feature branch:

```bash
git checkout -b feature/my-feature
```

3. Run tests:

```bash
cargo test
```

4. Submit a Pull Request.

## Contribution Guidelines

Please ensure:

* Code changes include appropriate tests.
* Security-sensitive changes are clearly documented.
* Cryptographic behavior is not changed without discussion.
* Public APIs remain backward compatible where possible.

## Development Branch

The `develop` branch is used for active development.

The `main` branch represents stable releases.
